Overview

Generating random variables from non-uniform distributions like the Gamma is computationally hard — no closed-form inverse CDF exists. This project reproduces the industrial-grade algorithms from Dieter & Ahrens (1974), the same paper behind R's internal rgamma() function.

The core insight: no single acceptance-rejection envelope can cover the full Gamma shape parameter domain a ∈ (0, ∞). The solution is a master routing function that dynamically dispatches to three specialized algorithms based on the value of a, each optimized for its range.

The Three Algorithms
// Algorithm GS
Piecewise Envelope
a ≤ 1
When a ≤ 1, the Gamma density has an asymptote at zero, making standard envelope construction impossible. GS bypasses this with a piecewise approach: a power-function proposal for the core region (0 ≤ x ≤ 1) and an exponential proposal for the tail (x > 1). Expected trials per accepted sample stays between 1.0 and 1.4 across all valid a.
// Algorithm GC
Cauchy Envelope
1 < a ≤ 2.53
Uses a Cauchy distribution as the majorizing envelope, centered at the Gamma's mode with scale chosen so the envelope touches the target at its peak. The threshold a = 2.53 is not a tuning choice — it is the exact value where the cubic discriminant in Lemma 3 becomes positive, below which GO's polynomial squeeze bounds are mathematically invalid.
// Algorithm GO
Normal-Exponential Split
a > 2.53
For large a, the Gamma approximates a normal. GO uses a Normal envelope for the core region and an Exponential envelope for the right tail, sampled with a fixed probability β = 0.009572 rather than computing the exact ratio at every call. Polynomial "squeeze bounds" from Lemma 3 allow 99%+ of proposals to be accepted or rejected without any expensive logarithm evaluations.

A Beta generator was also implemented via the Gamma-ratio theorem: if X ~ Gamma(a) and Y ~ Gamma(b) independently, then X/(X+Y) ~ Beta(a,b).

Verification — K-S Test Results

10,000 samples were generated for each algorithm and tested against the theoretical CDF using the Kolmogorov-Smirnov test. All p-values well above 0.05 — we fail to reject the null hypothesis that the samples match the theoretical distribution.

Key Insights
01
Algorithm boundaries are math, not parameters. The a = 2.53 threshold is the exact point where a cubic discriminant in Lemma 3 turns positive. Crossing it with the wrong algorithm produces subtly incorrect samples with no obvious error signal.
02
Squeeze bounds eliminate 99%+ of expensive operations. Instead of computing a full log-density ratio for every proposal, polynomial lower bounds accept or reject most samples in microseconds. The full evaluation is only reached for rare edge cases.
03
Statistical software is not monolithic. Functions like rgamma() are sophisticated routing wrappers, not single equations. Understanding the underlying proofs is what separates correct implementations from plausible-looking ones.
